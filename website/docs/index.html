Overview

Agent-2-Beta Open Source Last updated: 2026

Agent-2 is a self-hosted autonomous AI agent powered by Google Gemini — a coding assistant, terminal agent, security tester, and persistent memory system in one interface.

What is Agent-2?

Agent-2 gives you a real AI agent that runs on your own machine. It can write and run code, execute shell commands, scan networks, read and write files, search the web, and remember everything across sessions — all powered by Google Gemini with no cloud dependency beyond the API call.

It ships in two modes:

  • Web UI — Browser interface with workspaces, multi-tab terminals, Three.js 3D welcome, and real-time streaming.
  • CLI — Terminal-native agent (agent2cli.py) with the same brain, tools, and memory — works just like the Gemini CLI.

Key capabilities

  • 12 Agentic Tools — run_command, read_file, write_file, multi_edit_files, list_dir, grep_search, delete_file, scan_project, web_search, update_todo, save_memory, emit_plan
  • Workspaces — Claude Projects-style context management with per-workspace memory
  • Persistent Memory — Auto-extracts facts from every conversation; survives restarts
  • API Key Rotation — Up to 9 keys, auto-rotates on quota exhaustion
  • Security Testing — Autonomous vulnerability scanning, logic flaw detection (XSS/SQLi), nmap, metasploit workflows built in
  • Custom Providers — Bring any OpenAI- or Anthropic-compatible API as a selectable model
  • Burp Suite MCP — Expose every Burp tool (Proxy, Repeater, Intruder, Scanner) to the agent
✅
Free & Open SourceAgent-2-Beta is MIT licensed. Clone it, modify it, self-host it for free.
ℹ️
Requires Python 3.10+A Gemini API key is required (free tier available). All other dependencies are installed automatically by run.py.

Installation

Installation

5 min setupAll platforms

Agent-2 installs with a single command. The one-line installer downloads the repo, builds an isolated .venv, installs every dependency, and starts the app.

One-line install (recommended)

macOS / Linux
curl -fsSL https://raw.githubusercontent.com/aaravshah1311/Agent-2-Beta/main/install.py | python3 -
Windows PowerShell
irm https://raw.githubusercontent.com/aaravshah1311/Agent-2-Beta/main/install.py | python -

Prefer to clone manually? Follow the steps below.

Prerequisites

  • Python 3.10+ — required. Check: python --version
  • Git — for cloning
  • Gemini API key — free at aistudio.google.com/app/apikey

Install steps

  • 1
    Clone the repository
    Download Agent-2 from GitHub to your machine.

cd Agent-2-Beta

  • 3
    Paste your Gemini API key
    The wizard will prompt you. The key is saved to agent2.db.
  • 4
    Open the web UI

API Keys

API Keys

Agent-2 stores API keys in its agent2.db SQLite database. Multiple keys are supported and auto-rotated when one hits its quota.

Get a free API key

Gemini API keys are free. Visit aistudio.google.com/app/apikey — sign in with Google and create a key.

Add your first key (via run.py)

bash
agent2 --addapi

The wizard walks you through pasting a key interactively. You can add multiple keys for failover.

Add a key inside the CLI

cli
/addapi

Saves the key to agent2.db immediately. Active on the very next request.

Manage keys in the UI

Open Settings → Keys in the web UI to add, rename, pin, reset, or delete keys and watch per-key token usage live. You can store up to 9 keys, all auto-rotated on quota.

🔌
Bring your own modelBeyond Gemini, add any OpenAI- or Anthropic-compatible endpoint under Settings → Providers (or /provider add in the CLI). It appears in the same model dropdown.

How rotation works

  • Agent-2 always uses the first active key
  • If a key returns a 429 (quota exhausted), it's marked inactive and the next key is tried automatically
  • If all keys are exhausted, they all reset and the cycle starts over
  • You can manually pin a key in the web UI Settings → Keys panel
🔑
Check key status in the CLIType /keys to see all keys, their active status, and per-key token usage.

Workspaces

Workspaces

Every chat belongs to a workspace. The agent always knows your project path, framework, and accumulated memories for that workspace.

Create a workspace

  • 1
    Click + Create Workspace in the sidebar
    Opens the workspace creation modal.
  • 2
    Enter a name and optional project path
    Leave path blank → Agent-2 creates a workspaces/YourName/ folder automatically. Or browse with the built-in path browser.
  • 3
    Click the workspace to enter it
    Then create a new chat. Every chat inside a workspace inherits its context.

What the agent knows inside a workspace

  • The absolute path to the project folder
  • The detected framework, language, and package manager
  • All memories accumulated in this workspace
  • The last working directory for terminal commands

Web UI Guide

Web UI Guide

The web interface runs at http://localhost:1311. It combines a workspace sidebar, chat panel, and real terminal in one view.

Interface layout

  • Sidebar (left) — workspace list → click a workspace → chat list
  • Topbar — current workspace name, model/mode selector, token counter
  • Chat panel (centre) — conversations with tool call blocks and plan cards
  • Terminal panel (right) — multi-tab xterm.js terminals, drag-resizable

Terminal features

  • ↑ / ↓ arrow keys — navigate command history (up to 200 commands per tab)
  • Esc — clear the input field
  • ▶ run button on tool blocks — click to run that command in the active terminal
  • + Term — open a new terminal tab
  • ■ kill — first click sends SIGTERM; second click (or after 3s) force-kills with SIGKILL
  • stdin row — appears when a process is running; type to send input to interactive programs

Editing messages

Hover any of your messages → click the ✏️ pencil icon → edit the text → press Enter. Agent-2 deletes that message and everything after it, then re-runs from the edited point.

File attachments

Click 📎 in the input bar to attach code files, images, PDFs, or text. Text files are embedded verbatim; images and PDFs are sent as binary blobs to Gemini's vision API.

Stopping generation

Click the ⏹ red stop button (appears during generation) to cancel mid-flight. The agent checks for the stop signal before each API call and before running each command.

Start the CLI

.venv/bin/python agent2cli.py --mode thinking "design this architecture"

The prompt

prompt format
you [workspace|model|mode]> _

The prompt shows your active workspace, current model, and mode icon (⚡★🧠) at all times.

Memory in CLI

Memory is stored in ~/.agent2/memories.json and persists across all CLI sessions. Conversation history is stored in ~/.agent2/history.json.

Memory System

Memory System

Agent-2's memory engine extracts facts from every conversation and persists them across sessions — the agent gets smarter the more you use it.

Memory types

TypeScopeSource
globalAll conversationsAdded manually by user
workspaceOne workspace onlyAdded manually or by agent
autoGlobal or workspaceExtracted automatically after each response

Auto-extraction

After every agent response, a background thread calls Gemini to extract memorable facts (user preferences, project details, environment configs, etc.) and saves them with an importance score 1–10.

Manage memories

cli
/memory              # list all memories with importance scores
/addmem Python 3.12 is installed on this machine   # add manually

In the Web UI, open Mem in the sidebar footer → three tabs: Global / Workspace / Auto-learned.

Burp Suite MCP

Burp Suite MCP

Web SecurityMCP BridgePortSwigger

Agent-2 connects to Burp Suite's official MCP server and hands every Burp tool — Proxy history, Repeater, Intruder, Scanner, site map, active-scan issues, raw HTTP requests — straight to the AI agent. Ask in plain English; the agent drives Burp for you.

🕷️
What is MCP?The Model Context Protocol is an open standard that lets an AI agent call an external tool's functions. PortSwigger ships an official MCP Server extension for Burp; Agent-2 speaks to it over a local SSE connection and exposes each Burp capability as an agent tool named burp_*.

What you get

  • Proxy history access — the agent can read captured requests/responses and reason over your live traffic.
  • Send & replay requests — issue raw HTTP requests through Burp (Repeater-style) and read the response inline.
  • Intruder & Scanner — launch and read results from Burp's fuzzing and active/passive scans, subject to what your Burp edition exposes.
  • Site map & issues — pull the discovered site map and active-scan issue list to prioritise findings.
  • Every tool, automatically — whatever tools the MCP server advertises are discovered at connect time and offered to the model — no per-tool wiring.
  • Works in both agents — the same bridge powers the Web UI and the CLI, and is exposed to custom OpenAI/Anthropic providers too.
⚠️
Burp edition mattersThe Scanner and some Intruder features require Burp Suite Professional. Proxy history, site map, and request replay work on Community edition. Agent-2 shows exactly which tools your Burp exposes after connecting.

Step 1 — Install the MCP server in Burp

  • 1
    Open Burp → Extensions → BApp Store
    Search for “MCP Server” (published by PortSwigger) and click Install.
  • 2
    Open the new “MCP” tab
    Tick “Enabled”. Note the listening address — it defaults to 127.0.0.1 port 9876.
  • 3
    Leave Burp running
    Keep Burp open with the MCP server enabled while you use Agent-2.
💡
EndpointAgent-2 talks to the SSE endpoint at http://127.0.0.1:9876/sse. You can paste either the bare host (http://127.0.0.1:9876) or the full /sse URL — Agent-2 tries both.

Step 2 — Connect from Agent-2

Web UI

  • 1
    Open Settings → Burp
    Click the gear icon, then the Burp tab.
  • 2
    Confirm the URL and click Connect
    The status dot turns green and shows the number of tools discovered. A chip list of every Burp tool appears below.
  • 3
    (Optional) Enable Auto-connect
    Toggle Auto-connect so Agent-2 re-establishes the Burp bridge automatically on every turn. This preference is saved and shared with the CLI.

CLI

cli
/burp connect        # connect to Burp's MCP server
/burp status         # show connection state, URL, tool count
/burp list           # list every Burp tool now available to the agent
/burp disconnect     # drop the bridge

Step 3 — Use it

Once connected, just talk to the agent — it picks the right Burp tool on its own:

example prompts
show me the last 20 requests from Burp proxy history
replay the login POST to /api/session and show the response headers
find every request that set a Set-Cookie header
resend that request with the id parameter changed to 1 OR 1=1
what active-scan issues has Burp found for this host?
summarise the site map Burp has discovered so far

Configuration reference

SettingDefaultWhat it does
BURP_MCP_URLhttp://127.0.0.1:9876Base URL of Burp's MCP server (env var override).
BURP_MCP_ENABLED0 (off)Initial default for auto-connect if never toggled in the UI.
Auto-connect toggleoffWhen on, Agent-2 connects to Burp every agent turn. Persisted in agent2.db, shared by web + CLI.
Tool namespaceburp_*Every Burp tool is exposed to the model under a sanitised burp_ name.

Troubleshooting

ProblemSolution
“mcp package not installed”Run agent2 --reset (or pip install mcp inside the .venv) and retry.
Timed out connectingIs Burp running with the MCP Server BApp Enabled? Check the MCP tab’s address matches your URL.
Connected but 0 toolsRe-tick “Enabled” in Burp’s MCP tab, then /burp connect again.
Scanner tools missingActive Scanner requires Burp Suite Professional; Community exposes Proxy/site-map/replay only.
Wrong portSet BURP_MCP_URL or paste the correct http://127.0.0.1:PORT in Settings → Burp.
⚠️
Authorized testing onlyThe Burp bridge lets the agent send live traffic to targets. Only drive it against systems you own or are explicitly authorized to test.

Troubleshooting

Troubleshooting

Common problems and their solutions.

ProblemSolution
No API keys configuredRun agent2 --addapi or type /addapi in the CLI.
Key quota exhaustedKeys rotate automatically. Add more keys: agent2 --addapi
Model returns empty responseThe model tier may not support that model. Switch: /model 2.5-flash
Terminal not showing outputRefresh the browser tab. Socket connection may have dropped.
python not found on WindowsUse py run.py or install Python from the Microsoft Store.
Port 1311 already in useChange port=1311 in agent2web.py to another port.
Broken venv / import errorsRun agent2 --reset — wipes and reinstalls.
CLI spinner stuckCtrl+C — cancels the request cleanly and returns to prompt.
rich not installedagent2 --reset — rich is now in the install list.
Want to start completely freshagent2 --uninstall then agent2
agent2cli.py not foundPlace agent2cli.py in the same folder as run.py.
Thinking mode not workingThinking is only available on 2.5/3.1 models (2.5-flash, 2.5-pro, 3.1-flash, 3.1-pro).
